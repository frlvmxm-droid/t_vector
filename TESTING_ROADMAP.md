# Testing roadmap

## Текущий подход
- `test_all.py` остаётся smoke/regression-скриптом для быстрых ручных прогонов перед релизом.

## Новый подход для развития
- Все **новые** тесты добавляются в `pytest`-структуру (`tests/`).
- `test_all.py` не расширяется новой логикой; его роль — только регресс-дым.

## Этапы миграции
1. Добавлять pytest-тесты для новых фиксов/фич.
2. Постепенно переносить критичные сценарии из `test_all.py` в `tests/`.
3. Подключить линтер/типизацию в CI (ruff/mypy поэтапно).
4. После покрытия основных путей — оставить `test_all.py` как минимальный smoke.
